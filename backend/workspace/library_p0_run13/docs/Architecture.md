# 产品需求文档 (PRD) — 图书管理系统

## 1. 项目概述

开发一套图书管理系统，实现图书的借阅、归还、读者管理以及查询统计等核心业务流程，提升图书管理效率与用户体验。系统采用 B/S 架构，后端基于 Python FastAPI 构建 RESTful API，前端提供简洁易用的 Web 界面。

## 2. 用户角色

| 角色 | 描述 | 主要职责 |
|------|------|----------|
| 管理员 | 系统管理人员 | 录入/编辑/下架图书；管理读者信息；查看统计数据；处理异常借还 |
| 读者 | 图书借阅者 | 查询图书；借阅/归还图书；查看个人借阅记录 |

## 3. 核心功能列表

### 3.1 图书管理
- **图书录入**：管理员新增图书，包括书名、作者、ISBN、出版社、出版年份、分类、库存数量等字段
- **图书编辑**：管理员修改图书信息
- **图书下架**：管理员将图书标记为下架状态，下架图书不可借阅
- **图书查询**：支持按书名、作者、ISBN、分类等条件进行模糊搜索

### 3.2 读者管理
- **读者注册**：管理员录入或读者自助注册，记录姓名、证件号、联系方式等信息
- **读者信息维护**：管理员更新读者资料或冻结/解冻读者账号
- **读者查询**：按姓名、证件号等检索读者信息

### 3.3 图书借阅
- **借书操作**：管理员扫描/输入图书与读者信息，完成借出登记，系统记录借阅时间并自动计算应还日期（默认30天）
- **借阅限制**：每位读者最多同时借阅 5 册图书；逾期未还图书时不可再次借阅
- **借阅记录**：记录每笔借阅的图书、读者、借书时间、应还时间、实际归还时间

### 3.4 图书归还
- **还书操作**：管理员扫描/输入图书信息完成归还
- **逾期处理**：系统自动计算逾期天数并提示逾期信息
- **状态更新**：归还后图书库存自动恢复，读者借阅额度释放

### 3.5 查询统计
- **图书统计**：展示图书总数、各分类数量、当前借出数量
- **读者统计**：展示读者总数、活跃读者数
- **借阅排行**：热门图书借阅排行
- **逾期统计**：当前逾期借阅清单

## 4. 非功能需求

| 类型 | 要求 |
|------|------|
| 性能 | 常见操作（查询、借还）响应时间 < 2 秒；支持 50 名用户并发访问 |
| 可用性 | 系统 7x24 小时运行，计划内维护除外；Web UI 适配主流浏览器（Chrome / Edge / Firefox 最新版） |
| 安全性 | 管理员操作需登录验证；读者信息脱敏展示；防止 SQL 注入与 XSS 攻击 |
| 可维护性 | 后端代码遵循 PEP 8 规范；API 接口遵循 RESTful 设计；提供接口文档 |
| 数据一致性 | 借阅/归还操作需保证事务一致性，避免超借或库存负数 |
